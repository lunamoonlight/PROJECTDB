# Проект: База данных «Цветочный магазин»

![Изображение] (https://i.pinimg.com/736x/92/bc/9a/92bc9aa264a241f8c85a0d64a6955f61.jpg)

## О проекте
<p>Данный проект представляет собой реляционную базу данных, разработанную для автоматизации работы цветочного магазина. Система охватывает ключевые бизнес-процессы: управление клиентами, обработку заказов, контроль товарных запасов, взаимодействие с поставщиками и организацию доставки. База данных реализована на PostgreSQL и обеспечивает целостность данных через систему внешних ключей и нормализованную структуру.</p>

## Структура базы данных и связи между таблицами
### Сущности
- *Clients* - содержит информацию о клиентах магазина 
- *Card* - хранит данные о картах постоянных клиентов 
- *Suppliers* - учёт поставщиков цветов и сопутствующих товаров 
- *Products* - каталог товаров магазина 
- *Stock* - контроль остатков на складе 
- *Orders* - основная информация о заказах 
- *Order_Items* - состав заказов (связь многие-ко-многим между Orders и Products) 
- *Delivery* - информация о доставке заказов 

### Связи 
- **Clients ↔ Card** — один-к-одному
Каждый клиент может иметь не более одной карты лояльности.

- **Clients ↔ Orders** — один-ко-многим
Один клиент может совершать множество заказов.

- **Suppliers ↔ Products** — один-ко-многим
Один поставщик может поставлять множество товаров.

- **Products ↔ Stock** — один-к-одному
Каждый товар имеет одну запись на складе.

- **Orders ↔ Order_Items** — один-ко-многим
Один заказ может содержать несколько позиций товаров.

- **Products ↔ Order_Items** — один-ко-многим
Один товар может входить в состав разных заказов.

- **Orders ↔ Delivery** — один-к-одному
Каждый заказ может иметь одну запись о доставке.
